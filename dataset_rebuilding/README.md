# Dataset rebuilding — Step 1: re-caption the real images

Goal: replace the thin, generic IAB captions (~20-50 words) with **dense,
per-class captions** (40-80 words, diffusion-prompt style) generated by
**Qwen3.5-9B** (vision, via Ollama). These richer captions will later be used
to regenerate harder synthetic counterparts so the attribution task stops being
trivial.

This step only re-captions the **20 000 real images** (2 000 × 10 semantic
classes). Synthetic regeneration is a later step.

## Files

| File | Where it runs | What it does |
|------|---------------|--------------|
| `caption_prompts.py`        | —            | One captioning prompt per semantic class (edit these to tune style). |
| `caption_real_images.py`    | compute node | Ollama client: walks `real/`, captions every image, writes CSVs. Resumable. |
| `setup_ollama_cineca.sh`    | **login node** | Installs Ollama into `$WORK` and pre-pulls `qwen3.5:9b` (compute nodes have no internet). |
| `slurm_caption.sh`          | compute node | Starts `ollama serve` on the GPU, then runs the captioner. |

## Workflow on CINECA Leonardo

```bash
# 0. Sync the repo + dataset to $WORK (from your local machine), if not already:
rsync -avz /mnt/data3/rtrebiani/hyperbolic_CLIP/ \
    <user>@login.leonardo.cineca.it:$WORK/hyp_fine_tuning/hyperbolic_CLIP/
# (iab_dataset/ and iab_captions/ are already on $WORK per the training setup.)

# 1. ONE-TIME, on the LOGIN node (has internet): install Ollama + pull the model
bash dataset_rebuilding/setup_ollama_cineca.sh        # ~6.6 GB download

# 2. Submit the captioning job (1× A100, 6h wall-time)
cd $WORK/hyp_fine_tuning/hyperbolic_CLIP
sbatch dataset_rebuilding/slurm_caption.sh

# 3. If it hits the wall-time, just resubmit — it skips finished images.
sbatch dataset_rebuilding/slurm_caption.sh
```

## Output

New CSVs in `$WORK/iab_captions_detailed/`, **same filenames and schema** as the
original IAB caption dir, so it is a drop-in replacement:

```
COCO.csv                ImgPath,Caption
AnimalFace_cat.csv      ImgPath,Caption
...
imagenet-1k.csv         ImgPath,Label,Caption   (Label carried over by stem)
```

Captions are keyed by image-filename **stem**, matching `IABCLIPDataset`. To use
them later, point the loader at the new dir:
`IABCLIPDataset(captions_dir="$WORK/iab_captions_detailed", ...)`.

## Ollama version is pinned (do not bump blindly)

Leonardo's GPU driver is `535.274.02` (CUDA 12.2). **Ollama v0.30.x** ships CUDA
kernels that this driver cannot load → every model crashes on load with
`CUDA error: device kernel image is invalid`. We therefore pin **v0.24.0**, the
last release that runs on this driver *and* supports `qwen3.5`. It installs to
`$WORK/ollama-0.24.0` (the `slurm_caption.sh` and `setup_ollama_cineca.sh` point
there). To try a newer version, test `ollama run qwen3.5:9b` on an interactive
GPU node first and watch the serve log for that CUDA error.

`qwen3.5` is a **reasoning model**: by default it emits a long `Thinking…` chain
(~60 s for a trivial prompt). The captioner sends `"think": false` in every
request, which suppresses it (~0.4 s for the same prompt). Keep that.

## Tips

- **Check quality first**: smoke-test one image per class before the full run:
  ```bash
  python dataset_rebuilding/caption_real_images.py \
      --dataset_path $WORK/iab_dataset --output_dir /tmp/cap_smoke \
      --model qwen3.5:9b --smoke
  ```
  (Requires an Ollama server up — run it inside an interactive GPU session, or
  add `--smoke` to the SLURM command.)
- **Tune the prompts** in `caption_prompts.py` (per class) and re-run the smoke
  test until the style is what you want, *then* launch the full job.
- **Resumable**: progress is flushed per row; failed images aren't written and
  are retried on the next run. Use `--overwrite` to start a class from scratch.
- **Speed**: ~20k images on one A100 ≈ a couple of hours. To go faster raise
  `OLLAMA_NUM_PARALLEL` + `--num_workers` together, or shard across GPUs with a
  SLURM array (one class subset per task) — not needed at this scale.
